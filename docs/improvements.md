# Portfolio Improvements Backlog

## Prioritization Model
- **P0:** Must-fix for usability/credibility
- **P1:** Strong improvement with moderate effort
- **P2:** Nice-to-have polish

## P0 — High Priority

### 1) Strengthen project credibility with proof links
**Problem:** Some project descriptions are strong but lack immediate validation paths.

**Actions:**
- Add `GitHub` links for every featured project.
- Add `Live Demo` or `API Docs` where possible.
- Mark unavailable demos clearly (e.g., “private deployment, demo on request”).

**Success criteria:**
- Every project card has at least one proof link.

### 2) Improve measurable impact language
**Problem:** Some statements are descriptive but not measurable.

**Actions:**
- Replace generic claims with numbers where available:
	- latency reduction
	- model performance
	- throughput/concurrency
	- response-time improvements

**Success criteria:**
- Each experience/project entry includes at least one concrete metric when data is available.

### 3) Accessibility contrast verification pass
**Problem:** Theme combinations may drift as styles evolve.

**Actions:**
- Recheck contrast in dark and light themes for headings, tags, and muted text.
- Confirm keyboard focus visibility for all interactive controls.

**Success criteria:**
- No low-contrast text in major content blocks.

---

## P1 — Medium Priority

### 4) Add visual context to projects
**Problem:** Project area is copy-heavy for quick scanning.

**Actions:**
- Add lightweight project visuals (static image or diagram thumbnail).
- Keep images compressed and optional on small screens if needed.

**Success criteria:**
- Each featured project includes a visual aid without hurting performance.

### 5) Tighten skills section density
**Problem:** Skills cards can feel tall on smaller screens.

**Actions:**
- Reduce spacing and optimize card rhythm.
- Consider compact tag treatment where it improves scanning.

**Success criteria:**
- Skills section shows more signal above the fold on mobile.

### 6) Refine hero support visual
**Problem:** Hero code block may read as generic if not tied to personal value.

**Actions:**
- Keep the code visual, but tailor lines to actual strengths/projects.
- Ensure readability and contrast in both themes.

**Success criteria:**
- Hero visual reinforces brand instead of acting as generic decoration.

---

## P2 — Lower Priority

### 7) Reduce non-essential motion cues
**Actions:**
- Reevaluate scroll indicator prominence.
- Keep motion subtle and optional for reduced-motion users.

### 8) Add lightweight analytics (privacy-friendly)
**Actions:**
- Add anonymous pageview tracking for portfolio iteration insights.

### 9) Improve social share assets
**Actions:**
- Refresh OG image design with clearer personal branding.

---

## Suggested Execution Order
1. P0.1 Proof links
2. P0.2 Metrics update
3. P0.3 Accessibility pass
4. P1 visual/content polish
5. P2 optional enhancements
